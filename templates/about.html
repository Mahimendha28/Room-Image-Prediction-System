Project Flow

From foldered dataset images to live browser predictions.

Open Prediction

1. Data Collection

Images are organized into class folders: Bathroom, Bedroom, Dinning, Kitchen, and Livingroom.

2. Preprocessing

Each image is resized to a fixed size, converted to grayscale, and normalized for stable feature extraction.

3. Feature Extraction

HOG and color descriptors capture structure, edges, and color cues before feature selection reduces the final vector.

4. Model Training

The pipeline splits data, balances classes, then trains either a tuned SVM or a soft-voting ensemble.

5. Evaluation

Accuracy, precision, recall, F1-score, and confusion matrix are used to measure class-wise performance.

6. Inference API

Uploaded images are processed through the saved model and returned with confidence and top alternatives.

Checklist Coverage

EDA

Class counts, imbalance ratio, and underrepresented classes are computed from the dataset and shown in the app.

Feature Engineering and Selection

HOG and color features are engineered, then SelectKBest keeps the most informative subset for training.

Ensemble Methods

A soft-voting ensemble combines SVM, Random Forest, and Extra Trees for stronger final predictions.

Monitoring

Each prediction is logged with timestamp, label, confidence, and top matches for lightweight monitoring.
